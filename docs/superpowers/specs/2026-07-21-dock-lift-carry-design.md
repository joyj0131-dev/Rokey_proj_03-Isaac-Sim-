# 2로봇 순차 도킹 + 파지 + 리프트 + 운반 (Plan 3) — 설계

> 작성: 2026-07-21. 관련: `two-robot-fleet-design.md` v3(E2E 미션 3~5단계),
> `two_robot_carry_demo.py`(검증된 grip+lift+carry), Plan 1·2(주행·측위 완료).

## 목표 (사용자 정의)

> "인계장에 있는 차 밑으로 두 로봇이 **순서대로** 들어가서, **바퀴감지 없이
> 하드코딩된 정지 위치**에서 바퀴를 잡고, 들어올려서 이동한다. 이걸 눈으로 본다."

## 범위

**포함**: 2로봇이 실제 메카넘 구동으로 차량 하부에 순차 진입 → 하드코딩 파지
위치 정지 → 팔 전개로 바퀴 파지 → 리프트 → 짧은 거리 운반. 헤드리스 검증 + GUI 시연.

**제외**:
- 바퀴/뎁스 센서 감지 (사용자 명시적 제외 — 정지 위치는 차량 축 좌표에서 하드코딩)
- Plan 2의 도크→인계장 ArUco 주행과의 체이닝 (이미 동작. 통합은 후속)
- 차종 일반화 (A5_Coupe로 고정 — 2로봇 리프트 검증 차종)

## 핵심 재사용 (위험 최소화)

`two_robot_carry_demo.py`가 이미 검증한 것을 최대한 그대로 쓴다:
- **씬**: A5_Coupe + 로봇 2대 + 축별 파지 기하 (`build_test_stage`)
- **파지**: `ARM_TARGETS` + `set_arms(scale)` (팔 전개 각도, 검증됨)
- **운반**: `set_cmd_vel` 메카넘 구동 + carry 로직 (`CARRY_SPEED`, facing 부호)
- **측정**: `car_lift_m`(실제 상승), `carry_follow_ratio`, `grip_jitter_ratio`

**딱 한 가지를 바꾼다**: 차량 하부 진입을 `set_world_poses()` 순간이동
(현재 코드 line 658 "guided ingress") → **실제 메카넘 직진 구동**으로 교체.

## 아키텍처

새 스크립트 `two_robot_dock_lift_demo.py` (carry demo 뼈대 복제 후 진입부 교체):

```
build_test_stage()   # carry demo 재사용 + 콜라이더 수정 추가
  ├ A5_Coupe 배치 (검증된 위치·방향)
  ├ 로봇 2대: 차량 하부 밖 진입 시작점에 스폰 (순간이동 아님)
  └ 콜라이더 수정: A5_Coupe 바퀴 Collision.height → 시각 폭 (override)

run_demo()
  1. 순차 진입 (신규): robot_far 가 먼 축 밑으로 직진(vx>0) →
     하드코딩 grip_x 도달 시 정지. robot_near 는 robot_far 가 진입로를
     벗어난 뒤 출발(순차) → 자기 grip_x 정지. 회전 없음.
  2. 파지: set_arms 램프 (0→1) — 두 로봇 동시
  3. 리프트 판정: car_lift_m 측정
  4. 운반: 두 로봇 편대 구동 CARRY_DISTANCE_M 이동
  5. 리포트 JSON + 모션품질
```

### 진입 정지 위치 (하드코딩)

차량 축 좌표에서 계산 (carry demo가 이미 실제 휠 좌표에서 축 위치를 구함 —
그 값을 진입 목표로 쓴다). 로봇은 자기 grip 위치의 x/z/yaw 목표로 P제어 직진.
센서 감지 없음. 정지 = 목표 위치 도달(위치 오차 < 허용).

### 순차 진입 이유

두 로봇이 동시에 좁은 하부로 들어가면 진입 중 충돌·간섭. 먼 축 로봇이 먼저
들어가 자리잡은 뒤 가까운 축 로봇이 진입 → 진입로 공유 시간 최소화.

## 콜라이더 수정

A5_Coupe 바퀴 콜라이더가 시각 대비 1.61배 넓어(타이어 안쪽 간격 콜라이더 0.99m
vs 시각 1.18m, 로봇 폭 ~0.95m — 4cm 여유뿐) 진입이 빡빡하다. 조사 결과의 권장책:
**테스트 스테이지에서 각 바퀴 `Collision.height`를 시각 폭으로 override**
(원본 `fab_vehicles.usd` 비파괴, 구동계·서스펜션 raycast 무관). 수정 후 간격
1.18m → 로봇 0.95m가 0.23m 여유로 진입.

## 좌표/제약

- 진입은 **직진만** (제자리 회전 비신뢰 — Plan 2 실측). 로봇은 진입 방향을
  향한 채 스폰되어 vx 구동으로 직진.
- 로봇 에셋은 낮춘 바퀴 변형(`_lowered.usd`, 지상고 +3cm) 사용 — 하부 진입 여유 확보.
- GPU PhysX, 원본 USD 비파괴, 좀비 프로세스 정리 (기존 관례).

## 검증 (도착 좌표만 보지 않는다 — Plan 2 교훈)

리포트 JSON + 판정:
1. **리프트**: `car_lift_m ≥ 0.025 m` (실제 상승, carry demo 기준 재사용)
2. **운반**: `carry_follow_ratio ≈ 1.0` (차가 로봇 따라 이동), 실제 구동 거리 ≥ 목표
3. **파지 안정**: `grip_jitter_ratio` 낮음 (파지 중 차량 진동)
4. **모션품질**(`motion_quality_probe.py` 재사용): 진입·운반 중 두 로봇 발레·점프·
   이탈 없음. **이게 통과 못 하면 PASS 아님.**
5. **옆칸 차량 무변위**: 대상 외 차량 최대 변위 낮음 (기존 진단 지표)

## 테스트 계획

1. 헤드리스 1회 → 위 5개 판정 콘솔 + JSON 리포트.
2. GUI 시연 — 순차 진입·파지·리프트·운반 육안 확인 (사용자 관람).
3. 실패 시: 진입 못 함(콜라이더/기하) vs 리프트 안 됨(팔 기하) vs 운반 실패를
   리포트 지표로 구분해 원인 격리.

## 리스크

- A5_Coupe 리프트는 carry demo에서 순간이동 진입 기준 검증. **실제 구동 진입으로
  바꿨을 때 파지 정렬이 어긋날 수 있음** — 진입 정지 위치 정밀도가 관건. 하드코딩
  좌표 + P제어 정밀도로 대응, 안 맞으면 정지 위치 튜닝.
- 순차 진입 타이밍(먼 축 로봇이 얼마나 들어가야 가까운 축 로봇 출발) — 실측 조정.
- 낮춘 바퀴(+3cm)가 carry demo의 검증 조건과 다름 — 리프트 재현 재확인 필요.
